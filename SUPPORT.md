# 支持说明

## 1. 仓库定位

本仓库是创业技术底座中的 Vue 3 管理后台模板，默认提供登录、工作台、动态菜单、RBAC 与用户 / 角色 / 菜单管理能力。

该仓库提供模板级支持，适合快速搭建后台基座，不承诺生产 SLA。

## 2. 提问前建议先检查

- [README.md](./README.md)
- [CONTRIBUTING.md](./CONTRIBUTING.md)
- [SECURITY.md](./SECURITY.md)
- [RELEASE.md](./RELEASE.md)
- `.env.*`、`vite.config.ts`、`src/api`、`src/router`

## 3. 适合提交公开 Issue 的场景

- 登录、刷新 Token、动态路由、权限渲染等主流程可复现异常
- 文档、示例接口约定或页面行为与实际实现不一致
- 通用后台能力存在明显缺口，且适合作为模板沉淀
- 构建、类型检查或依赖升级后出现兼容性问题

## 4. 不建议提交公开 Issue 的场景

- 安全漏洞、越权路径、敏感接口或生产环境令牌泄露
- 带有公司内部地址、账号、Cookie、真实客户数据的调试信息
- 只适用于单一业务项目的专属定制诉求

涉及安全或敏感内容时，请改走 [SECURITY.md](./SECURITY.md)。

## 5. 提问时建议包含的信息

- Node.js 与 npm 版本
- 本地运行还是容器 / 代理环境
- 最短复现步骤
- 页面表现、控制台报错、网络请求结果
- 使用的后端接口约定是否与模板保持一致

## 6. 响应优先级

- 可复现 bug 优先
- 影响主流程的文档错误优先
- 通用能力增强会结合整体模板路线统一评估

## 7. Fork 使用建议

如果你将本模板用于自有项目，建议同步调整支持渠道、响应窗口和环境信息模板，以便团队成员更快定位问题。
