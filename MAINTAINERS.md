# 维护者说明

## 1. 当前维护角色

- 仓库所有者：`gee-coder`
- 默认维护范围：后台模板结构、通用组件、路由权限、文档与 CI

fork 后建议改为你自己的维护者名单、评审责任人与发布负责人。

## 2. 需要重点评审的变更

- 登录状态、刷新令牌与请求拦截逻辑
- 动态菜单、权限码与路由守卫行为
- 全局布局、导航模型与基础 UI 规范
- 核心依赖大版本升级

## 3. 合并标准

- `npm run type-check` 与 `npm run build` 通过
- 页面或接口行为变更同步更新文档
- 新增通用能力优先复用现有目录与模式
- 避免把业务强耦合逻辑直接沉淀进模板

## 4. 分支与版本建议

- 稳定分支：`master`
- 功能分支：`feat/*`
- 修复分支：`fix/*`
- 发布标签：`vX.Y.Z`

## 5. 发布责任

发布者需要确认：

- `CHANGELOG.md` 已补齐
- `RELEASE.md` 的检查清单已执行
- 后端接口契约变化已在 README 或相关文档中说明
