# 发布流程

## 1. 版本建议

建议使用语义化版本：

- `MAJOR`：破坏性结构或接口契约变更
- `MINOR`：新增向后兼容的模板能力
- `PATCH`：缺陷修复、文档完善、构建调整

## 2. 发布前检查清单

### 必做检查

- 更新 [CHANGELOG.md](./CHANGELOG.md)
- 核对 README、环境变量说明与页面截图 / 预览说明
- 确认接口契约与后端保持一致

### 本地校验命令

```powershell
npm ci
npm run type-check
npm run build
```

## 3. 建议的发布步骤

1. 合并改动到 `master`
2. 补充 `CHANGELOG.md`
3. 运行类型检查与生产构建
4. 创建版本标签，例如 `v0.1.0`
5. 在 GitHub Release 中记录新增能力、修复内容与升级提示

## 4. 发布说明建议包含

- 本次新增页面或能力
- 与后端接口约定的变化
- 需要同步调整的环境变量
- 已知限制与兼容性说明
