# 贡献指南

本仓库是后台管理模板，重点是保持页面分层、API 隔离与权限模型一致性。

## 1. 基本原则

- 所有接口请求统一放在 `src/api`
- 组件内不直接拼 URL
- 页面复杂逻辑优先拆到子组件或可复用逻辑
- 权限与菜单相关逻辑尽量复用现有 store 与路由映射工具

## 2. 推荐开发流程

1. 从最新 `master` 拉取
2. 创建特性分支
3. 本地开发并联调后端
4. 运行类型检查与构建
5. 提交代码并发起 PR

## 3. 提交前检查

```powershell
npm run type-check
npm run build
```

## 4. 页面与组件建议

- 单个 `.vue` 文件尽量控制在易读范围内
- 表格页优先拆为查询区、表格区、表单抽屉
- 新增模块时，优先同步补 `src/types`
- 动态路由能力不要改成散落在页面里的硬编码

## 5. PR 检查清单

- 是否新增或修改了 API 契约
- 是否更新了页面文档或 README
- 是否破坏现有登录、刷新、权限流程
- 是否保持移动端基础可用性
